# Apuntes DDS - UTN FRBA - 1C 2026

Apuntes de **Desarrollo de Software** en Markdown.

## Por dónde empezar

1. Leé `Apuntes_Desarrollo_de_Software.md`.
2. Usá el mapa inicial para ir clase por clase.
3. Cerrá cada clase con el resumen mental y el checklist de examen.
4. Usá `study/` para practicar y extender preguntas, flashcards o mapas conceptuales.

## Qué incluye

```txt
Apuntes_Desarrollo_de_Software.md  # apunte principal integrado
docs/
  metodologia-apuntes.md           # criterio pedagógico usado
  indice-clases.md                 # estado de clases y fuentes de referencia
study/
  preguntas/                       # espacio para cuestionarios
  flashcards/                      # espacio para tarjetas de repaso
  mapas-conceptuales/              # espacio para diagramas
  glosario.md                      # glosario vivo
```

## Cómo colaborar

El branch `main` está protegido para mantener estable el apunte.

Si encontrás una mejora, error o tema para discutir:

- abrí un **issue** para proponer cambios, reportar dudas o marcar inconsistencias;
- abrí un **pull request** si ya tenés una corrección concreta;
- evitá subir material que no sea propio o que no corresponda compartir.

La idea es que el repo sea colaborativo, pero con trazabilidad: cada cambio importante debería tener contexto y revisión.

## Nota sobre fuentes

Los apuntes fueron elaborados a partir de clases, material de cátedra, tomas propias y repos públicos de ejemplo. Los derechos del material original pertenecen a sus autores y a la cátedra correspondiente.

Este repo publica una síntesis de estudio, no una redistribución del material fuente.
